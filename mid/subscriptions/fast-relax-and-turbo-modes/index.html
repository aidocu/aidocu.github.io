快速、放松和涡轮模式

Midjourney 使用强大的图形处理单元 (GPU) 来解释和处理每个提示。 当您购买 Midjourney 订阅时，您就购买了这些 GPU 上的时间。

不同的订阅计划具有不同的每月 GPU 时间量。
此每月订阅的 GPU 时间是快速模式时间。 快速模式尝试让您立即访问 GPU。 它是默认处理层，并使用您订阅的每月 GPU 时间。

我的生成需要多少 GPU 分钟？​

Midjourney 机器人处理的平均作业需要大约一分钟的 GPU 时间才能完成图像创建。 使用非标准长宽比或旧的 Midjourney 模型版本放大图像可能需要更多时间。 创建变体或使用较低质量值将花费更少的时间。

作业的时间取决于以下因素：

+ 较低时间++ 平均时间+++ 较高时间
作业类型变体/imagine放大
长宽比默认（方形）自定义高或宽
质量参数--q 0.25 或 --q 0.5默认值 (--q 1)--q 2 （对于旧模型版本)
停止参数--stop 10---stop 99默认（--stop 100）

在运行进程之前和之后使用 /info 可以查看该生成使用了多少剩余 GPU 分钟。

快速模式与放松模式​

Standard、Pro 和 Mega 计划的订阅者每月可以在放松模式下创建无限数量的图像。 放松模式不会消耗任何 GPU 时间，但作业将根据您使用系统的情况放入队列中。

在放松模式下我需要等待多长时间？​

放松模式下的作业被放置在队列中，以便在 GPU 可用时进行处理。 Relax 的等待时间是动态的，但通常每个作业的等待时间在 0 到 10 分钟之间。 如果您偶尔使用“放松”模式，与经常使用该模式的订阅者相比，您的等待时间将会更短。 目前，每当您续订每月订阅时，此优先级都会重置。

局限性​

使用 Relax 模式时，排列提示、--repeat 参数和 Legacy upscaler、Max Upscale 不可用。

快时间用完了？​

您可以在 Midjourney.com/accounts 页面上购买更多 Fast Hours。

默认情况下，图像是使用快速模式生成的。
每月未使用的 Fast GPU 时间不会滚动。
续订时，快速模式会自动重新激活。

涡轮模式​

Turbo 模式适用于想要极快生成图像的用户。 Turbo 模式使用高速实验 GPU 池。 在 Turbo 模式下运行的作业生成速度提高了四倍，但消耗的订阅 GPU 分钟数是典型快速模式作业的两倍。

Turbo 模式仅适用于 Midjourney 模型版本 5、5.1、5.2 和 6。

如果选择 Turbo 模式，但 GPU 不可用，或者与所选型号版本不兼容，您的作业将以 Fast 模式运行。

Turbo 模式是一项实验性功能，可用性和价格可能随时发生变化。

如何切换模式​

使用 /fast /turbo 或 /relax 命令​

Standard、Pro 和 Mega 计划订阅者可以使用 /fast 或 /relax 命令在模式之间切换。

使用设置命令​

使用 /settings 命令并从菜单中选择 🐢 Relax 🐇 Fast 或 ⚡ Turbo。

使用 --fast --turbo 或 --relax 参数​

Standard、Pro 和 Mega 计划订阅者可以在提示末尾使用 --relax、--fast 或 --turbo 参数，以使用 Relax、Fast 或 Turbo 模式运行单个作业，而不用更改其首选设置。
